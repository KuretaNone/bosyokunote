## rbtnn

Introducing the TabPanel Feature: Motivation and Implementation

### about me 

- name rbtnn(なる)
- se マネジメント 仕事ではほとんどコード書かなくなった
- vim をソースコードからビルドして使ってる
- IDE等は苦手

### What's TabPanel

- ほかのエディタでは1ファイル1タブ
- vim では windowのつまり、1ファイルではないということ
- vim ではtabPageって読んでね
- tabPanelとは縦タブみたいなやつ
- 9.1.1391にてvimにマージされた。
- vim onlyでNeovimには採択されていない
- vimでtabを使っている人が少ないのでは？

### Motivation と 背景

1. vim を理解しながら開発ができることがモチベ
2. 既存のtabLineが狭すぎる
3. Windowの管理がめんどくさい。tabpanelだとその管理をしなくていい。

### Design
TabLineとTabPanelで共存したかった。vimの機能に影響したくなかった。
改行をいれた情報を表示したかった。

### 実装について
- 左側に固定して実装する
- Col, + TabPanel_widthで幅を与える
- vim は CUI,GUIマルチOS 分あるのでその修正が大変
- PopUpWindowを重ねて表示するのはあきらめる
- popupのタイミングでは描画タイミングをちょうせいする必要がある
- GUIのデバッグは難しい
- vimは行で要素をもっているからスクロールすると、バッファ側も移動して大変
- これ以上自分できれいに書く方法がわからなかったから、とりあえずPullリクをおくったらあっさりマージ
- 上のような状況んおとき、どうすればいいかわからない
- 自分だけがほしい機能（かもしれない）時にどうすればいいのかPRを送る都岡

### Usage & Configuration

- TabPanelのオプションに改行を入れたかった
- g:actuial_curtabpage
- 改行を入れたら複数行で表示できる機能を実装できてよかった
- ShowTabpanel
- TabPanelopt 複数設定するときは, で区切る
    - align 表示位置
    - colums 幅
    - vert 区切リ銭をだすかどうか
    - vert

### Q&A
- Q. TabPanelとテストとドキュメントはどうしてたのか
- A. テストはむずかしかったが、ドキュメントの量としてはそんなになかった。
- Q. 5年くらいPRを寝かしていたが、コンフリクトはおきなかったのか
- A. 2,3ヶ月くらいに一回起きる。ブラムさんが健在だったの時に、リファクタリングした時が一番大変だった。
- Q. nvimにマージされなかった理由の分析は？
- A. nvim ユーザではないので、わからないし、そこまでの熱意はなかった・
